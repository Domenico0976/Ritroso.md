# Standard File Header — Ritroso.md v3

> This file defines the standard header that every file in the set
> must have at the top. The specific content of each file is in the
> individual templates in `prompts/file-set-templates/`.

---

## Standard Header (mandatory for every file)

```markdown
---
name: "{file_name}"
description: "Dynamic guide for the agent reading {file_name}. Interpret this file as the operative lens for {file_objective}; use the current project context, inferred constraints, declared limits, anti-patterns, and unresolved questions to choose the safest and most context-coherent approach. If explicit context is missing, prefer inference grounded in linked files and surface remaining uncertainty as [OPEN] instead of hallucinating."
---

# {NUMBER}_{NAME}.md — {project_name}
> Generated by: Ritroso.md v3 skill
> Domain: {domain_slug}
> Status: [DRAFT | RITROSO-VERIFIED]
> Date: {generation_date}
> Folder: new-ideas/{domain_slug}/{project_name_slug}/
> Inference loop: [PENDING | COMPLETE | {n} resolutions, {m} open]
> Set: [[00_INDEX]] [[01_GOAL]] [[02_PRODUCT]] [[03_NEXT_STEPS]]
>      [[04_ELEMENTS]] [[05_COMPONENTS]] [[06_PRICE]] [[07_BUDGET]]
>      [[08_LIMITS]] [[09_AGENTS]] [[10_ERROR]] [[11_INTERPOLATION]] [[12_ASKED]]
---
```

## Description Rules

The `name` field must always match the real file name exactly.

The `description` field must be generated dynamically for EACH file and must:

1. State the file's operative purpose in the current project context
2. Describe the best interpretation strategy for the agent reading it
3. Include inferred limits, declared constraints, anti-patterns, or uncertainty when relevant
4. Mention the file objective explicitly
5. Be specific to the domain and project, never generic if context exists
6. Fall back to cross-file inference if context is partial
7. In empty context, instruct the agent to rely on linked files and unresolved questions

## Dynamic Description Formula

```text
{name} = exact file name

description =
"Use this file to reason about {file_objective} for {project_name} in the {domain_slug} domain.
Prioritize {primary_focus}; interpret decisions using {best_available_context}.
Respect these inferred or declared limits: {limits_summary}.
Avoid these anti-patterns: {anti_pattern_summary}.
Optimize for {outcome_target}.
If context is incomplete, infer only from linked files and keep unresolved points explicit."
```

## General Compilation Rules

1. **Mandatory wikilinks** in every file — pointing to the other 12 files in the set
2. **Justified N/A** — if a section does not apply: `N/A: [reason]`
3. **Free format** — the model uses the fastest format for itself
4. **No duplication** — if content is already elsewhere: `→ see [[file]]`
5. **[OPEN]** for every unverified assumption or missing information
6. **[CORRECTION]** for every correction applied during the negative prompt
7. **[INF]** for every value derived from the inference loop (not stated in prompt)
8. **[ASKED]** to flag content that came from a context gap question answer
